How I keep experiment logs

2026.06.29 4 min

An experiment log preserves judgment before it becomes a results table. More important than a single score is the data, code state, and reason that produced it.

Run conditions

Keep the model name, data path, preprocessing version, seed, and key hyperparameters in the same place. When possible, keep the exact command as well. 

python train_model.py \
  --model xlm-roberta-base \
  --data-dir data_no_iemocap \
  --seed 42

Observations

Scores should go into a table, but the log should also keep phenomena that numbers do not explain. Note whether one fold dropped sharply, whether validation loss was unstable from the start, or whether warnings appeared in the logs.

Next action

When the run ends, define the next condition immediately. If no further experiment is needed, write down why. Without this step, the log becomes a storage place for results and the research decision goes back into memory.
