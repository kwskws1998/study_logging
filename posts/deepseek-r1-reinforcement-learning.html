DeepSeek-R1: Reinforcement Learning for Reasoning

2026.06.30 14 min

This note follows the DeepSeek-R1 review deck and keeps only the English substance. Korean duplicate sections are folded into the same English explanation. A few fatal wording issues are corrected: GRPO is described as the RL algorithm used for DeepSeek-R1-Zero and DeepSeek-R1, the policy ratio is treated as a probability ratio, and the KL term is framed as a reference-policy regularizer rather than a vague mode-collapse fix.

Paper anchor: pure reinforcement learning, rule-based rewards, poor readability, and language mixing, language consistency reward, estimating the advantages from group scores.

Motivation

DeepSeek-R1 is about incentivizing reasoning behavior in large language models using reinforcement learning. The headline result is not just benchmark performance; it is that long chain-of-thought style behavior, reflection, verification, and strategy switching can emerge under an RL objective when the task has reliable verifiers.

DeepSeek-R1-Zero starts from DeepSeek-V3-Base and applies RL directly, without an initial supervised fine-tuning stage for reasoning traces. DeepSeek-R1 then adds a more structured multi-stage pipeline to fix readability, language mixing, and broader alignment issues.

DeepSeek-R1-Zero

The R1-Zero experiment tests a sharp claim: reasoning ability can be induced with RL alone on verifiable tasks. The model receives reward from final-answer correctness and output format, not from human-written reasoning trajectories. This makes the experiment useful because it tests whether the model can discover its own reasoning strategy instead of copying a supervised chain-of-thought style.

The tradeoff is that R1-Zero has rough behavior: its reasoning can be hard to read, and it can mix languages. This is why the later DeepSeek-R1 pipeline introduces cold-start data, more SFT, and another RL stage.

GRPO

GRPO stands for Group Relative Policy Optimization. The main practical difference from PPO is that GRPO removes the learned value model. Instead of training a separate critic to estimate expected return, it samples a group of outputs for the same question and normalizes each output's reward relative to the group.

\[ A_i = \frac{ r_i - \operatorname{mean}(\{r_1,r_2,\ldots,r_G\}) }{ \operatorname{std}(\{r_1,r_2,\ldots,r_G\}) } \]

Intuitively, an answer has positive advantage if it scores higher than the other sampled answers to the same question. It has negative advantage if it is worse than the group average. This is not a global quality score; it is a within-question relative score.

GRPO Objective

GRPO still uses a PPO-style clipped surrogate objective. For each sampled output \(o_i\), define the probability ratio:

\[ \rho_i(\theta) = \frac{\pi_{\theta}(o_i \mid q)} {\pi_{\theta_{\mathrm{old}}}(o_i \mid q)} \]

The ratio is not simply "the policy". It is the new policy's probability of the output divided by the old rollout policy's probability of the same output.

\[ \mathcal{J}_{\mathrm{clip}}(\theta) = \min\left( \rho_i(\theta)A_i,\, \operatorname{clip}(\rho_i(\theta),1-\epsilon,1+\epsilon)A_i \right) \]

If \(A_i > 0\), the model is rewarded for increasing the probability of that output, but clipping prevents the increase from becoming too large. If \(A_i < 0\), the model is pushed away from that output, again with a clipped trust region.

KL Regularization

GRPO also subtracts a KL regularization penalty so the trained policy does not drift too far from a reference policy:

\[ \mathcal{J}_{\mathrm{GRPO}}(\theta) = \mathbb{E}\left[ \frac{1}{G}\sum_{i=1}^{G} \left( \mathcal{J}_{\mathrm{clip}}(\theta) - \beta D_{\mathrm{KL}}(\pi_{\theta}\,\|\,\pi_{\mathrm{ref}}) \right) \right] \]

The paper uses the following estimator for the KL term:

\[ D_{\mathrm{KL}}(\pi_{\theta}\,\|\,\pi_{\mathrm{ref}}) = \frac{\pi_{\mathrm{ref}}(o_i \mid q)} {\pi_{\theta}(o_i \mid q)} - \log \frac{\pi_{\mathrm{ref}}(o_i \mid q)} {\pi_{\theta}(o_i \mid q)} - 1 \]

The deck's intuition is basically right, but the safer wording is this: the KL penalty regularizes policy drift from the reference model and helps preserve the base model's linguistic competence. Calling it only a mode-collapse fix is too loose.

Rewards for R1-Zero

R1-Zero uses two rule-based reward types:

  • Accuracy reward: checks whether the final answer is correct. For math, this can be verified with a boxed or otherwise structured answer. For coding, a compiler and hidden test cases can supply objective feedback.
  • Format reward: enforces that reasoning and final answer are placed in the expected tags, such as <think> and <answer>.
\[ R_{\mathrm{rule}} = R_{\mathrm{acc}} + R_{\mathrm{format}} \]

The important point is that the reward is verifier-based. The paper deliberately avoids neural reward models for the reasoning tasks because neural reward models can be hacked during large-scale RL.

Training Template

The R1-Zero template only constrains structure: reason first, answer last. It does not impose a specific reasoning style or content. That is why the observed self-reflection behavior matters. It suggests that the behavior was encouraged by the RL setup rather than hand-coded as a fixed reasoning script.

Self-Evolution and the Aha Moment

During RL, R1-Zero's average response length increases. The paper interprets this as the model allocating more thinking time to reasoning tasks. More importantly, the model starts to show reflection-like behaviors, including revisiting an earlier step, checking a possible mistake, and trying another solution path.

The "aha moment" is not a separate algorithmic component. It is an observed behavior in an intermediate checkpoint: the model appears to pause and reconsider its own reasoning. The correct interpretation is evidence of emergent reflective behavior under RL, not proof that the model is conscious or that it was explicitly programmed to reflect.

DeepSeek-R1 Pipeline

DeepSeek-R1 fixes the practical problems of R1-Zero with a four-stage training pipeline:

  1. Cold start: collect and refine a small amount of long CoT data to make the model readable and human-aligned enough to start RL.
  2. Reasoning-oriented RL: apply the same general RL idea as R1-Zero, while adding a language consistency reward.
  3. Rejection sampling and SFT: sample multiple answers from the best checkpoint, keep correct and readable reasoning traces, mix them with non-reasoning data, and fine-tune the original base model.
  4. RL for all scenarios: use rule-based rewards for reasoning data and model-based rewards for general helpfulness and harmlessness.

Cold Start

Cold start data is used because R1-Zero's raw outputs can be difficult to read. The authors collect long CoT examples, enforce a readable pattern with a final summary, and use human post-processing to clean the data. This is a practical compromise: pure RL demonstrates that reasoning can emerge, but the product-quality model still benefits from a small amount of curated initialization.

Language Consistency Reward

R1-Zero can mix languages, especially because the base model is trained heavily on Chinese and English. R1 introduces a language consistency reward:

\[ R_{\mathrm{language}} = \frac{ \operatorname{Num}(\mathrm{Words}_{\mathrm{target}}) }{ \operatorname{Num}(\mathrm{Words}) } \]

The tradeoff matters: this improves readability and alignment with the target language, but the paper reports a slight performance degradation in ablation results.

Rejection Sampling and SFT

After the first RL stage, the best checkpoint generates multiple candidate responses for each prompt. Rejection sampling keeps outputs that are correct and readable. The paper then uses about 600K reasoning samples and 200K non-reasoning samples for SFT.

This is one reason the "R1 is just pure RL" summary is incomplete. R1-Zero is the pure-RL demonstration; R1 is a multi-stage system that combines cold-start data, RL, rejection sampling, SFT, and a final RL stage.

Final RL Stage

The final RL stage mixes reasoning and non-reasoning prompts:

\[ R = R_{\mathrm{reasoning}} + R_{\mathrm{general}} + R_{\mathrm{language}} \]

Reasoning data still uses rule-based rewards. General data uses reward models. Helpfulness is judged from the final summary, while harmlessness is evaluated over the whole response, including the reasoning process, because harmful content can appear during intermediate reasoning.

Distillation

DeepSeek-R1 also acts as a teacher for smaller open-source models. The distilled models are fine-tuned on 800K samples generated by DeepSeek-R1. The paper applies SFT only for distillation and does not include an RL stage for those distilled models.

The deck's example is correct: DeepSeek-R1-Distill-Qwen-7B reports 55.5 on AIME 2024 pass@1, exceeding QwQ-32B-Preview's 50.0 in the table.

Corrected Q&A

Ratio and advantage in GRPO: \(A_i\) says whether an answer is better or worse than the other sampled answers for the same question. The ratio \(\rho_i\) says how much the new policy changes the probability of that same answer relative to the old policy. The clipped objective increases probability for positive-advantage answers and decreases probability for negative-advantage answers, while limiting large policy shifts.

DeepSeek-V3, R1-Zero, and R1: DeepSeek-V3-Base is the base model. R1-Zero is trained from that base model with GRPO-style RL and rule-based rewards, without an initial SFT stage for reasoning traces. R1 adds cold-start data, reasoning RL, rejection sampling plus SFT, and a final RL stage for broader helpfulness and harmlessness.

KL term: it is not the reward itself. It is a regularizer that subtracts a penalty when the policy drifts too far from the reference policy.

Takeaway

The important lesson is that verifier-based RL can elicit reasoning behavior from a strong base model, especially on tasks where correctness is easy to check. GRPO makes large-scale RL more practical by removing the value model and using group-relative rewards. DeepSeek-R1 then turns the raw R1-Zero result into a more usable model through cold start data, language consistency, rejection sampling, SFT, and final preference-oriented RL.

Source: DeepSeek-AI, DeepSeek-R1: Incentivizing Reasoning Capability in LLMs via Reinforcement Learning.
