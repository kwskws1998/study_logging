Auto-Encoding Variational Bayes: Variational Autoencoder Notes

2026.06.30 13 min

This note follows the VAE review deck: autoencoder, variational autoencoder, encoder outputs, reparameterization, decoder reconstruction, KL divergence, ELBO, and a compact implementation sketch. The core intuition is kept, but the mathematically fatal sign and notation errors are corrected.

Paper anchor: probabilistic encoder, probabilistic decoder, regularizer, element-wise product.

Autoencoder Baseline

A standard autoencoder compresses an input \(x\) into a latent representation and reconstructs it as \(\hat{x}\). The encoder learns a deterministic code, and the decoder learns to reconstruct the original input from that code.

\[ h = f_{\mathrm{enc}}(x), \qquad \hat{x} = f_{\mathrm{dec}}(h) \]

The training signal is usually reconstruction error. For a simple continuous vector example:

\[ \mathcal{L}_{\mathrm{recon}} = \sum_{j=1}^{d}(x_j-\hat{x}_j)^2 \]

What Changes in a VAE?

A variational autoencoder does not encode an input into one fixed latent vector. It encodes the input into a distribution over possible latent vectors. In the common diagonal Gaussian setup, the encoder predicts a mean vector and a log-variance vector:

\[ q_{\phi}(z \mid x) = \mathcal{N}\left(z;\mu_{\phi}(x), \operatorname{diag}(\sigma_{\phi}^{2}(x))\right) \]

This is the first important correction to the deck: the encoder should be described as outputting \(\mu\) and \(\log\sigma^2\). It should not say \(\log\mu^2\). The standard deviation \(\sigma\) is derived later from the log variance.

Encoder Output

If the hidden representation is \(h\), the VAE encoder usually has two linear heads:

\[ h = W_{\mathrm{enc}}x + b_{\mathrm{enc}} \]
\[ \mu = W_{\mu}h + b_{\mu}, \qquad \ell = W_{\log\sigma^2}h + b_{\log\sigma^2} \]

Here \(\ell = \log\sigma^2\). The slide label "sigma vector" is acceptable only as a shorthand after conversion. Technically, the encoder head should be the log-variance head, because this is the numerically stable parameter that the network predicts.

Why Log Variance?

Predicting variance directly can create invalid negative values. Predicting log variance avoids that problem because exponentiation maps any real number to a positive variance.

\[ \sigma^2 = \exp(\ell), \qquad \sigma = \sqrt{\exp(\ell)} = \exp(0.5\ell) \]

The wording should be "exponentiate log variance", not "multiply log variance by an exponential function".

Reparameterization Trick

Sampling \(z\) directly from \(q_{\phi}(z \mid x)\) would place the random sampling operation in the gradient path. The reparameterization trick separates the noise from the learned parameters:

\[ \epsilon \sim \mathcal{N}(0,I), \qquad z = \mu + \sigma \odot \epsilon \]

The \(\odot\) matters: this is elementwise multiplication. In code terms, sample standard normal noise with the same shape as the standard deviation, then shift and scale it by \(\mu\) and \(\sigma\). 

std = torch.exp(0.5 * logvar)
eps = torch.randn_like(std)
z = mu + std * eps

Decoder and Reconstruction

After sampling \(z\), the decoder maps it back toward the data space:

\[ \hat{x} = f_{\theta}(z) \]

The exact reconstruction term depends on the output distribution. For real-valued data, an MSE-like Gaussian reconstruction loss is common. For binary image data, a Bernoulli decoder usually corresponds to binary cross entropy.

ELBO

The VAE objective is usually written as an evidence lower bound. Using the common notation where the encoder is \(q_{\phi}(z \mid x)\) and the decoder or generative model is \(p_{\theta}(x \mid z)\):

\[ \log p_{\theta}(x) \ge \mathcal{L}(\theta,\phi;x) = \mathbb{E}_{q_{\phi}(z\mid x)} [\log p_{\theta}(x \mid z)] - D_{\mathrm{KL}}\left(q_{\phi}(z \mid x)\,\|\,p(z)\right) \]

This is the second important correction: the encoder is \(q_{\phi}\), and the decoder or generative model is \(p_{\theta}\). The deck's appendix flips these symbols in places.

Training Loss

Training often minimizes the negative ELBO. That converts the objective into reconstruction loss plus a positive KL regularizer:

\[ \mathcal{J}(\theta,\phi;x) = -\mathbb{E}_{q_{\phi}(z\mid x)} [\log p_{\theta}(x \mid z)] + D_{\mathrm{KL}}\left(q_{\phi}(z \mid x)\,\|\,p(z)\right) \]

For a diagonal Gaussian approximate posterior and a standard normal prior \(p(z)=\mathcal{N}(0,I)\), the KL term used in the loss is:

\[ D_{\mathrm{KL}} = \frac{1}{2}\sum_{j=1}^{J} \left(\mu_j^2 + \sigma_j^2 - \log\sigma_j^2 - 1\right) \]

Equivalently, if \(\ell_j = \log\sigma_j^2\):

\[ D_{\mathrm{KL}} = -\frac{1}{2}\sum_{j=1}^{J} \left(1+\ell_j-\mu_j^2-\exp(\ell_j)\right) \]

Critical Sign Correction

The deck writes this value as:

\[ \frac{1}{2}\sum_j \left(1+\log\sigma_j^2-\mu_j^2-\sigma_j^2\right) = -0.56882 \]

That expression is the \(-D_{\mathrm{KL}}\) term inside the ELBO. It is not the positive KL penalty used in the minimized loss. Therefore, the final loss should add \(+0.56882\), not \(-0.56882\).

\[ \mathcal{J} = 7.298611 + 0.56882 = 7.867431 \]

So the slide's conclusion \(6.729791\) is a sign error if the section is describing final training loss.

Implementation Sketch

The core implementation is short. The key is that the model predicts \(\mu\) and \(\log\sigma^2\), samples \(z\) through reparameterization, and adds a positive KL term to the reconstruction loss. 

def reparameterize(mu, logvar):
    std = torch.exp(0.5 * logvar)
    eps = torch.randn_like(std)
    return mu + std * eps

def vae_loss(x, x_hat, mu, logvar):
    recon = torch.nn.functional.mse_loss(x_hat, x, reduction="sum")
    kl = -0.5 * torch.sum(1 + logvar - mu.pow(2) - logvar.exp())
    return recon + kl

Takeaway

The deck's intuitive story is mostly usable: AE learns a compressed code, VAE learns a distribution around a code, reparameterization makes stochastic sampling trainable, and the objective balances reconstruction quality against a latent-space regularizer. The fatal corrections are the encoder output notation, the log-variance conversion wording, the \(q_{\phi}\) and \(p_{\theta}\) roles, and the KL sign in the final loss.

Source: Kingma and Welling, Auto-Encoding Variational Bayes.
